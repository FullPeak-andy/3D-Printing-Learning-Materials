4️⃣ 安全加固（拿 Root 后必做）

🔒 安全加固清单

拿到 Root 权限后，以下安全措施强烈建议执行：

# === KLP1 安全加固脚本 ===

# 1. 禁止 SSH 远程 Root 登录（推荐！）
sudo sed -i 's/^PermitRootLogin.*/PermitRootLogin no/' /etc/ssh/sshd_config
sudo systemctl restart sshd

# 2. 添加 SSH 密钥（如果没有的话）
ssh-copy-id mks@KLP1_IP

# 3. 改 SSH 端口（可选，security through obscurity）
sudo sed -i 's/^Port 22/Port 2222/' /etc/ssh/sshd_config
sudo systemctl restart sshd

# 4. 安装 fail2ban 防暴力破解
sudo apt update && sudo apt install -y fail2ban
sudo systemctl enable fail2ban

# 5. 设置防火墙（只开必需端口）
sudo apt install -y ufw
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow 80    # HTTP
sudo ufw allow 443   # HTTPS
sudo ufw allow 2222  # SSH（新端口）
sudo ufw enable

# 6. 定期检查登录日志
sudo tail -f /var/log/auth.log | grep -i sshd

⚠️ 警告：改 SSH 端口后，新端口必须记住！否则无法 SSH 登录。建议同时保留一个 USB 串口连接作为备用。

5️⃣ 决策树：现在要不要拿 Root？

🌲 Root 权限决策树

▼ 你现在需要 Root 吗？
否 → 暂时不需要
是 → 你要做什么？
装 Obico / 第三方插件？
→ 用方法 1，设密码
临时调试系统问题？
→ 用方法 2，sudo su -
想完全脱离 KINGROON？
→ 用方法 3，刷 Armbian

💡 实用建议：

  • 大多数 KLP1 用户不需要 Root：改 printer.cfg / 装 Moonraker 插件都不需要
  • 只有装 Docker / 改系统服务 / 刷固件时才需要
  • 建议先试方法 2（sudo su -），用完 exit 退出，不留持久化 Root
