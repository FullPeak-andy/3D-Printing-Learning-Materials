必修G29 加 ACCEPT + SAVE_CONFIG

📍 [gcode_macro G29] · 段中部
▼

问题：Z 轴校准值不写盘 → 每次重启都丢。

❌ 原版（你改错过的）
# Klipper 会试图执行 "ACCEPT + SAVE_CONFIG" 当一个命令 ACCEPT + SAVE_CONFIG
✅ 注释版（3 行分开）
PROBE_CALIBRATE ACCEPT SAVE_CONFIG
💡 为什么必须拆 3 行？
Klipper 解析器是按行执行的，写成 ACCEPT + SAVE_CONFIG 会被当成一个未定义命令 "ACCEPT + SAVE_CONFIG" → 报错 shutdown。拆成 3 行，每行一个独立命令。
3

必修G30 用 SAVE="default" + SAVE_CONFIG

📍 [gcode_macro G30] · 段中部
▼

问题：床面 mesh 校准数据每次写到一个新名字 "name"，default profile 还是旧数据，回零时 LOAD=default 加载的也是旧数据。

❌ 原版（你之前的）
BED_MESH_PROFILE SAVE="name" BED_MESH_PROFILE LOAD="name"
✅ 注释版（直接覆盖 default）
BED_MESH_PROFILE SAVE="default" SAVE_CONFIG
💡 原理动画

必修G30 删掉 LOAD="name" 死代码

📍 [gcode_macro G30]
▼

问题：测完 mesh 后又 LOAD="name"，但 SAVE_CONFIG 没写盘 = 内存里有，硬盘上没有 → 重启就丢。

BED_MESH_PROFILE LOAD="name" # 死代码：默认就是 default，不写这句也是 default
💡 LOAD=default 的作用
[homing_override] 末尾有 BED_MESH_PROFILE LOAD=default，这是回零后自动加载 default profile 的关键。G30 本身不需要再 LOAD 一次，删掉死代码即可。

推荐PID 4 行风格统一为 "key: value"

📍 [extruder] / [heater_bed]
▼

问题：Mainsail 编辑器显示 4 行红色警告（你之前问过）。

❌ 原版（Klipper 自动生成）
control = pid pid_kp = 19.837 pid_ki = 4.723 pid_kd = 20.829
✅ 注释版（统一风格）
control: pid pid_kp: 19.837 pid_ki: 4.723 pid_kd: 20.829
💡 改 : 风格后
Mainsail 红色警告消失，Klipper 解析器两种写法都接受，不影响功能。但末尾 #*# 块是 Klipper 自动生成的，没法改，红色忽略即可。

推荐verify_heater 阈值收紧

📍 [verify_heater extruder] / [verify_heater heater_bed]
▼

问题：原版阈值太松（max_error 120 / check_gain_time 120），温度大波动也不报警 = 假阴性。

推荐pressure_advance 0.03→0.04

📍 [extruder]
▼

问题：0.03 偏小，拐角处容易堆料；0.04-0.06 是 KLP1 默认挤出机的推荐区间。

pressure_advance: 0.03 # 偏小 pressure_advance: 0.04 # 推荐
💡 怎么进一步调到最佳？
打印一个 pressure advance 标定塔，从 0.02 打到 0.08，看哪个层角最锐利就选哪个。

推荐[homing_override] 加 LOAD=default

📍 [homing_override] 末尾
▼

问题：回零后不加载床面 mesh = 打印头走到 (X,Y) 时不会自动 Z 补偿，喷嘴可能撞到翘起的床面。

# [homing_override] gcode 块末尾加： {% if printer.bed_mesh.profiles.default %} BED_MESH_PROFILE LOAD=default {% endif %}

可选新增 [exclude_object] 对象排除

📍 段尾 · 对象排除
▼

切片时勾选"按对象排除" → 打印某个对象失败可以跳过它继续打别的。OrcaSlicer / PrusaSlicer 都支持。

💡 实战场景
一次打 3 个模型，A 失败 → 控制台 EXCLUDE_OBJECT_START NAME=A → 跳过 A 继续打 B+C，节省 1 小时。

可选PURGE_NOZZLE 坐标移出 mesh

📍 [gcode_macro PURGE_NOZZLE]
▼

擦嘴片位置 (X=10, Y=210) 在 mesh 范围外（mesh 是 X=20-202, Y=20-190），不会触发 Z 补偿 → 喷嘴不会"忽上忽下"。

📍 [verify_heater extruder]
▼

120 秒太长 = 温度已经飙到危险值才报警，60 秒更早检测到异常加热。

📍 [heater_bed]
▼
