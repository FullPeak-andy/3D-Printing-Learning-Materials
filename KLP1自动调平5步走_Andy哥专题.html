🎬 先分清三种"调平"

Andy 哥你问的"自动调平"其实有 3 层意思，先搞清楚才能对症下药：

层级 做什么 工具 频率
Leveling（平台调平） 让床面整体水平，4 个角 Z 偏移一致 CR-Touch / 手动拧螺丝 每次换床 / 移动后
Tramming（车床校准） 4 个角 + 中心，让喷嘴到床距离都≈一张纸 CR-Touch 自动 1 周 1 次
Mesh（床面网格） 把床面分成 N×N 网格，测每个点的 Z 偏移 Klipper bed_mesh 每次打印前
💡 关键概念： KLP1 出厂带的 CR-Touch（或者 BL-Touch）是个电容式探头，Klipper 用它自动测每个网格点的 Z 偏移。完全不用手拧螺丝，机器自己测。

🚀 自动调平一句话流程

① 硬件：CR-Touch 探头接好 X 限位（出厂已接）

② Klipper 配置：printer.cfg 加 [bltouch] + [bed_mesh] section

③ 自动校准：Mainsail 跑 BED_MESH_CALIBRATE 命令

④ 切片器：OrcaSlicer 起始 G-code 加 G29 或 BED_MESH_PROFILE LOAD=default

⑤ 保存：测完 SAVE_CONFIG，每次打印自动加载

⚠️ 你的 KLP1 出厂默认是 Marlin 固件 + 手动调平。后来刷成 Klipper 后自动调平需要重新配置。如果是新机器第一次用，强烈建议先看完 Tab 2-4 再动。

🔧 硬件准备 · 30 分钟检查

先确认 CR-Touch 探头在位 + Klipper 配置正确。硬件没毛病才能跑自动调平。

1
检查 CR-Touch 探头

看 KLP1 喷嘴左边那个蓝色小方块（CR-Touch），看探针能不能正常伸缩。

  • 探针亮蓝灯 = 正常
  • 探针亮红灯 = 触发状态（贴着床）
  • 灯不亮 = 没通电，检查 3 针线（白/红/黑）接 X 限位接口
💡 KLP1 出厂 CR-Touch 是接在 X-STOP 接口（3 根线：信号/VCC/GND），不能接到普通 IO。
2
SSH 登录 KLP1

用 MobaXterm / PuTTY 登录 KLP1（Klipper 主机）

# 主机名（最常用） ssh pi@klp1.local # 或者用 IP（路由器后台查 KLP1 的 IP） ssh pi@192.168.x.x # 默认密码（KLP1 官方是 raspberry 或 klp1） password: raspberry
3
查 printer.cfg 关键配置

登录后看 printer.cfg 里 CR-Touch 和 bed_mesh 部分：

cat ~/printer.cfg | grep -A 30 "bltouch\|bed_mesh"

如果有输出就跳过 Tab 3；如果没看到 [bltouch] 或 [bed_mesh]，去 Tab 3 加配置。

4
测试探针自检

在 Mainsail Console 跑：

BLTOUCH_DEBUG COMMAND=pin_down # 探针伸出 BLTOUCH_DEBUG COMMAND=pin_up # 探针收回 QUERY_PROBE # 查探针状态

输出 TRIGGERED 表示探针触发了（正常）；open 表示没触发（探针线松了）。

5
清床面 + 装好耗材
  • 用酒精擦床面（KLP1 默认 PEI 弹簧钢床）
  • 装好 PETG 耗材，确认能正常挤出（手动挤出 50mm 看效果）
  • 关上打印机门（防气流影响）

🖥️ Klipper bed_mesh 完整配置

把下面三段配置加到 KLP1 的 printer.cfg 末尾。复制粘贴就行，KLP1 出厂 Klipper 模板就是这三段。

1
编辑 printer.cfg
cd ~ nano printer.cfg

在文件末尾追加（如果已经有 [bltouch] 就跳过）：

2
追加 [bltouch] 配置
# ===== CR-Touch 探头配置（KLP1 标配）===== [bltouch] sensor_pin: ^PG12 # KLP1 主板 CR-Touch 信号引脚 control_pin: PB1 # 探针控制引脚 x_offset: -42 # 探针到喷嘴的 X 偏移（mm） y_offset: -10 # 探针到喷嘴的 Y 偏移（mm） z_offset: 2.55 # 探针触发到喷嘴尖距离（首次估个值） probe_with_touch_mode: True pin_up_touch_mode: True pin_up_reports_not_triggered: True stow_on_each_sample: True
⚠️ x_offset / y_offset / z_offset 三个值要按你的实际机器微调。KLP1 出厂手册有标称值，差异在 ±2mm 内都算正常。
3
追加 [bed_mesh] 配置
# ===== bed_mesh 床面网格配置 ===== [bed_mesh] speed: 120 # 探针移动速度（mm/s） horizontal_move_z: 5 # Z 抬升高度（mm） mesh_min: 15, 15 # 网格起点（X, Y） mesh_max: 225, 225 # 网格终点（X, Y） probe_count: 5, 5 # 5×5 网格（共 25 点） algorithm: lagrange # 插值算法（lagrange 最准） fade_start: 0.6 # 0.6mm 开始淡出补偿 fade_end: 10.0 # 10mm 完全淡出 mesh_pps: 2, 2 relative_reference_index: 12 # 中心点 12（5×5 的中心）
💡 mesh_pps: 2, 2 让 Klipper 在两个点之间再插 2×2=4 个虚拟点，打印出来的曲面更平滑。
4
追加 SAFE_HOMING + 起始 macro
# ===== 安全归位（避免喷嘴撞床角）===== [safe_z_home] home_xy_position: 125, 125 # 归位点（床中心） speed: 100 z_hop: 10 # ===== 一键自动调平 macro ===== [gcode_macro AUTO_BED_MESH] gcode: M117 开始自动调平 G28 # 归位 BED_MESH_CALIBRATE # 测网格 G1 Z30 F600 # 抬 Z M117 调平完成
5
保存 + 重启 Klipper
# nano 里：Ctrl+O 保存，Ctrl+X 退出 sudo systemctl restart klipper # 检查 Klipper 状态 sudo systemctl status klipper

看到 active (running) 绿色就 OK。Mainsail 刷新页面，应该能看到"自动调平"按钮。

📐 OrcaSlicer 校准按钮使用

你截图顶部"校准"按钮就是 OrcaSlicer 自带的校准向导，能调 5 样东西：

📍 校准 → 床形（Z 偏移 / 床面形状） ← 你问的"自动调平"是这个

OrcaSlicer 校准 → 床形功能会引导你做两件事：

步骤 1：手动 Z 偏移（必做）

  1. 点"校准"→"床形"→"Z 偏移"
  2. OrcaSlicer 提示你在喷嘴下放一张 A4 纸
  3. 点"开始"，打印头移到床中心 Z=0
  4. 用纸在喷嘴和床之间摩擦（喷嘴刚好擦纸，能感觉到阻力）
  5. Mainsail 控制台输入 SET_GCODE_OFFSET Z=+0.05（每次 0.05 调整）
  6. 调到纸能滑动但有阻力

⏱ 5 分钟

步骤 2：床面网格（可选）

如果你在 Tab 3 加了 bed_mesh 配置，这步可以跳过——Klipper 会在打印前自动跑 BED_MESH_CALIBRATE。

如果要用 OrcaSlicer 引导：

  1. 点"校准"→"床形"→"自适应网格"
  2. 它会发 BED_MESH_CALIBRATE 给 Klipper，KLP1 自动跑 5×5 网格
  3. 完成后点"保存到 Klipper"
📍 校准 → 温度（温度塔） ← 你现在在做的

你正在用！步骤见之前发的 PETG 温度塔完全指南。重点：

  • 起始温度 260°C，结束 220°C，步长 5°C（9 段）
  • 先解决 Tab 1 那个功率 off 问题再打印
📍 校准 → 流量（流量系数 / E 步进）

挤出流量校准，调出料量。回收 PETG 直径不均，强烈建议做。

  1. 校准 → 流量 → 流量系数
  2. OrcaSlicer 让你画 100mm 线 + 测量剩余
  3. 输入实际挤出长度，自动算系数
📍 校准 → 压头（压力提前 PA）

Klipper 特有：压力提前补偿，打印转角时减速。

  1. 校准 → 压头 → 压力提前
  2. OrcaSlicer 印一个 L 形 + 圆弧测试件
  3. 看转角有没有拉丝/鼓起
📍 校准 → 加速度

最大加速度测试。KLP1 默认 1500 mm/s² 偏激进，PETG 建议调到 800-1200。

🎯 推荐校准顺序（Andy 哥）

  1. 手动 Z 偏移（先做这个，否则后面全白做）
  2. 床面 mesh 测一次 + SAVE_CONFIG
  3. 温度塔（已经在做）
  4. 流量系数（回收 PETG 必做）
  5. 压力提前（看转角质量）
💡 Andy 哥做 LED 灯罩/外壳，Z 偏移 + 床面 mesh 是两个最关键的——直接影响灯罩透明度（PETG 透光率 80%+，层纹明显就会糊）。

🩺 排错与微调

自动调平失败或结果不对的 5 个常见问题：

❌ 探针碰到床后报错 "Probe triggered before movement"

原因：Z_offset 值太大，探针还没降到底就触发。

解决：printer.cfg 里减小 z_offset（如 2.55 → 2.4），重启 Klipper。

❌ 自动调平后第一层还是粘不住 / 太近

原因：手动 Z 偏移没调准，paper test 没做。

解决：

  1. Mainsail 控制台 SET_GCODE_OFFSET Z=0 重置
  2. 喷嘴到床中心 Z=0，放纸
  3. 微调 Mainsail 的 BABYSTEPPING Z（±0.025 mm 步进）
  4. 调到纸能滑动但有阻力
❌ mesh 测到一半报错 "BLTouch failed to deploy"

原因：探针线松了或电磁干扰。

解决：

  1. 关打印机重开（重启 Klipper）
  2. 检查探针 3 针线
  3. printer.cfg 加 pin_up_reports_not_triggered: False
❌ 床面 mesh 范围不够（4 角粘不住）

原因：mesh_min / mesh_max 设置太保守。

解决：printer.cfg 改：

mesh_min: 10, 10 mesh_max: 230, 230

接近 KLP1 最大打印范围（220×220）。

❌ 每次打印前都要等调平（浪费时间）

解决：

  1. OrcaSlicer 起始 G-code 改用 BED_MESH_PROFILE LOAD=default（加载已保存的 mesh，不重新测）
  2. 只在"明显不平"时手动跑 BED_MESH_CALIBRATE
  3. printer.cfg 加 relative_reference_index: 12 用中心点归一化
# OrcaSlicer 起始 G-code 替换（Andy 哥优化版） G28 # 归位 BED_MESH_PROFILE LOAD=default # 加载上次 mesh G1 X10 Y10 Z5 F6000 # 移出料线位置 G29 # 触发 Klipper 自动 mesh（如果启用）
