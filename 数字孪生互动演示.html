📝 docker-compose.yml 核心配置

  1. 创建项目目录 ~/klipper-digital-twin
  2. 编写 docker-compose.yml（Mosquitto + Node-RED + InfluxDB + Grafana 四个服务）
  3. 配置 Mosquitto（开启 TCP 1883 + WebSocket 9001）
  4. 执行 docker-compose up -d 启动
  5. 浏览器验证 4 个服务端口（1883/1880/8086/3000）
🌐 服务端口速查：
  • MQTT: 1883（TCP）/ 9001（WebSocket）
  • Node-RED: http://<IP>:1880
  • InfluxDB: http://<IP>:8086
  • Grafana: http://<IP>:3000（admin / admin123）

🌉 Moonraker → MQTT 桥接

这是连接 Klipper 和整个数字孪生系统的"翻译官"

  1. 创建 Python 虚拟环境 python3 -m venv venv
  2. 安装依赖 pip install paho-mqtt websockets
  3. 编写桥接脚本 bridge.py（订阅 Moonraker WebSocket）
  4. 创建 systemd 服务 /etc/systemd/system/moonraker-mqtt.service
  5. 启动 systemctl start moonraker-mqtt

❓ 常见问题 FAQ

Q：Node-RED 连不上 InfluxDB？
A：容器内用 http://influxdb:8086，不是 localhost。
Q：MQTT 收不到消息？
A：检查 Mosquitto 端口 1883 是否开放；用 mosquitto_sub -h localhost -t "klipper/#" -v 命令测试。
Q：Grafana 查询无数据？
A：时间范围选择 "Last 5 minutes"；检查 Bucket 名是否正确（区分大小写）。
Q：桥接脚本频繁断连？
A：增加 WebSocket 心跳；在 systemd 配置 RestartSec。
Q：性能开销大不大？
A：全套服务约占 树莓派 4B 内存 800MB-1.2GB，CPU 5-10%，可接受。
